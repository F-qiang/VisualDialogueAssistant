# AI 视觉对话助手

选题：题目一 AI 视觉对话助手

## 一、项目简介

本项目是一款面向“AI 视觉对话助手”赛题的多模态对话应用，支持摄像头画面接入、麦克风语音输入、语音识别、视觉内容理解、多轮对话与语音播报。系统运行后，用户可以直接对着设备提问，系统将结合当前画面、语音内容和上下文信息生成自然、连贯、可追问的回复。

本项目重点关注以下几个方面：

- 视觉内容理解是否准确、稳定。
- 语音交互是否自然、流畅。
- 端侧与云端协同是否合理。
- 是否能够有效控制模型调用成本。
- 是否满足赛事对原创性、持续提交与文档完整性的要求。

## 二、技术依赖

### 2.1 界面层

- `PySide6`

### 2.2 采集层

- 摄像头采集与图像处理：`opencv-python`、`numpy`
- 音频采集与静音检测：`sounddevice`

### 2.3 云端 AI 服务

- ASR 语音识别：百度智能云短语音识别 / 讯飞开放平台语音听写
- VLM 多模态视觉：智谱清言多模态 API / 通义千问多模态 API
- LLM 文本对话：智谱清言文本模型 / 通义千问文本模型
- TTS 语音合成：百度智能云 TTS / 讯飞开放平台 TTS

### 2.4 工具依赖

- 网络请求：`requests`
- 环境变量管理：`python-dotenv`
- 图片处理辅助：`Pillow`
- 日志记录：`logging`

## 三、原创功能说明

本项目的原创功能主要包括：

1. 四层分层架构设计与状态机管控。
2. 端侧静音检测、画面变化检测与输入过滤逻辑。
3. 规则化意图路由，按需分发模型请求。
4. 多轮对话上下文管理与摘要压缩机制。
5. 全流程异常兜底与分级提示体系。

## 四、项目设计特点

### 4.1 按需触发视觉分析

系统不会对每一帧画面都调用大模型，而是在用户主动提问或检测到明显画面变化时才进行视觉分析，从而降低成本。

### 4.2 规则优先的意图路由

对于纯文本问题和视觉相关问题，系统会先通过轻量规则进行分类，再决定调用文本模型还是多模态模型。

### 4.3 多轮上下文控制

系统仅保留最近几轮完整对话，旧对话会进行摘要压缩，以减少上下文长度并保持回答连贯。

### 4.4 端侧前置过滤

通过静音检测、画面变化检测、语音置信度判断等方式，尽量在本地过滤无效输入，减少无意义请求。

## 五、运行说明

1. 授予程序摄像头和麦克风权限。
2. 确保网络正常，因为部分能力依赖云端 AI 服务。
3. 安装项目依赖后启动程序。
4. 启动后可直接通过语音向 AI 发起提问。
5. 若遇到摄像头画面不清晰或环境噪声较大，可调整位置后重试。
