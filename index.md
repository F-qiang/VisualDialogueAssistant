## 二、代码结构总览

### 1. 根目录
- `main.py`：程序入口。
- `README.md`：项目概览、技术依赖、运行说明、当前进度与文档入口。
- `index.md`：项目总索引，负责统一展示文档导航、代码结构和当前进度。
- `requirements.txt`：Python 依赖清单，用于快速安装运行所需库。
- `.gitignore`：Git 忽略规则，避免提交缓存文件、临时文件和敏感信息。
- `.env.example`：环境变量示例文件，用于说明 API Key、地址和运行参数的占位配置。

### 2. `app/` 应用级配置与状态
- `app/__init__.py`：标记 `app` 为 Python 包，便于模块导入。
- `app/config.py`：项目配置中心，管理应用名称、窗口尺寸、摄像头编号、录音参数、API 地址等全局配置。
- `app/state.py`：运行状态定义与状态管理，记录系统当前是否处于空闲、录音中、识别中、推理中或播报中。
- `app/prompts.py`：提示词模板管理，存放系统角色设定、回复风格要求、上下文摘要提示等。

### 3. `core/` 核心业务流程
- `core/__init__.py`：标记 `core` 为 Python 包。
- `core/camera.py`：摄像头采集模块，负责打开摄像头、读取画面、截取关键帧并提供给界面或推理层使用。
- `core/audio.py`：音频采集模块，负责麦克风录音、静音检测、语音结束判定和音频片段保存。
- `core/vision.py`：视觉处理模块，负责图像压缩、画面变化检测、亮度判断和视觉请求前置处理。
- `core/router.py`：意图路由模块，判断用户输入是否需要视觉信息，并决定调用文本模型还是多模态模型。
- `core/context.py`：上下文管理模块，负责保存最近多轮对话、裁剪历史、生成摘要和支持上下文清空。
- `core/pipeline.py`：主流程编排模块，串联摄像头、音频、识别、路由、模型调用和输出流程。
- `core/response.py`：回复生成与格式整理模块，负责把模型输出整理为适合界面展示和语音播报的结果。

### 4. `services/` 云端服务封装
- `services/__init__.py`：标记 `services` 为 Python 包。
- `services/asr_client.py`：ASR 服务调用封装，负责将录音文件发送给语音识别服务并解析返回文本。
- `services/llm_client.py`：文本大模型调用封装，负责处理纯文本对话、上下文摘要和兜底回复。
- `services/vlm_client.py`：多模态视觉模型调用封装，负责上传图像、文字和上下文并获取视觉问答结果。
- `services/tts_client.py`：语音合成服务调用封装，负责将文本转换为音频文件并供界面播放。

### 5. `ui/` Qt 界面层
- `ui/__init__.py`：标记 `ui` 为 Python 包。
- `ui/main_window.py`：主窗口实现，负责布局摄像头预览区、对话区、状态区和控制区。
- `ui/widgets.py`：可复用界面组件，封装文本显示控件、状态提示控件和操作按钮等通用部件。

### 6. `utils/` 通用工具层
- `utils/__init__.py`：标记 `utils` 为 Python 包。
- `utils/logger.py`：日志工具，统一输出运行日志、错误日志和调试信息。
- `utils/image_utils.py`：图像工具，负责图片格式转换、压缩、编码和基础处理。
- `utils/audio_utils.py`：音频工具，负责音频格式处理、采样率转换、静音辅助判断等。
- `utils/error_handler.py`：异常处理工具，负责统一封装报错提示、日志记录和兜底逻辑。
